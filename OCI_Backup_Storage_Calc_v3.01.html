📖 How Retention Works — สูตรการคำนวณ
Retention Logic
  • DAILY: Incremental copies = dailyRet × dailyChangeRate × effectiveFullSize
  • WEEKLY: Full copies = weeklyRet × effectiveFullSize
  • MONTHLY: Full copies = monthlyRet × effectiveFullSize (ramp-up month by month)
  • YEARLY: Full copies = Capped at yearlyRet (เพิ่มทุกสิ้นปี)
  • TOTAL = Daily + Weekly + Monthly + Yearly
  • Standard Tier = Total - ArchiveClassTier | ArchiveClassTier = Backups ที่เก่ากว่า Archive after months
Input: Compression % = 0–99 (e.g. 30 = 30%)
Input: Daily Change % = 0–100 (e.g. 3 = 3% per day)

compRatio = 1 ÷ (1 − Compression % / 100)
effectiveSize = rawGB ÷ compRatio ÷ dedupRatio

Key rules for OCI Managed Services:
(1) Dedup = 1.0x for all OCI native services — OCI Object Storage has no deduplication.
(2) ADB backup storage cost is included in service pricing up to 60-day retention; no separate Object Storage charge.
(3) OCI PostgreSQL and MySQL HeatWave: backup algorithm is OCI-controlled and not user-configurable.
(4) BDS / Exadata: user can configure RMAN compression level — BASIC requires no additional license on OCI.
(5) All ratios are vs actual used data (not provisioned/allocated disk size). Always ask customer for actual data usage, not disk size.
